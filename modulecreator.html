Créer son propre module

I. Coder le module

Créer un fichier python avec le nom exact du module. Ce fichier python doit obligatoirement contenir une fonction execute() qui possède un argument "arg" de type liste.

Vous devez ensuite ajouté un fichier nommé "version.txt" avec uniquement la version inscrite. Exemple : 1.2.3

Vous pouvez ajouté un fichier "authorisation.txt" qui contiendra les autorsations nécessaires au module. Exemple : displaysplit rep

Liste des autorisations possibles qui seront transmis en argument lors de l'éxécution du module :

  • displaysplit : transmet sous la forme d'un int le choix de l'utilisateur d'afficher ou non sa réponse séparé par des espaces
  • adresse : transmet sous la forme d'un str l'adresse du dossier de l'utilisateur
  • rep : transmet sous la forme d'un str la requête complète de l'utilisateur
    •

    Vous n'êtes pas obligés d'utiliser l'argument "arg" mais il doit obligatoirement être présent dans la déclaration de la fonction execute()

    Les autorisations sont séparés par des espaces et écrit sur une seule ligne.

    CmdOS ira chercher en priorité les autorisations inscrites dans le fichier authorisation.txt. S'il n'et pas présent, il ira les chercher dans la base de donnée.

    Pour indiquer que vous n'avez pas besoin d'autorisations, écrivez simplement "NULL" dans ce fichier.

    Si nécessaire, vous pouvez ajoutez d'autres fichiers.

    Il est fortement conseillé d'inclure dans le fichier principal la création d'un dossier dans le dossier utilisateur pour les fichiers créés par le module. Par contre, utilisez le dossier du module pour les fichiers temporaires.

    II. Envoyer le module sur GitHub

    Créer vous un compte ou connecter-vous.

    Créer un dépôt et mettez-y tous les fichiers de votre module avec le fichier python principale à la racine du dépôt.
