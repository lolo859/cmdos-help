Godmode

Le godmode est un emsemble de fonction réservé aux développeurs et utilisateurs aguerri.

Il faut être en mode admin pour y accéder.

Voici la liste des fonctions du godmode :

  • sys.variables : Cette fonction permet d'afficher la valeur des variables systèmes

  • sys.cmd.store.install.vanish_loading : Cette fonction permet de masquer les étapes de l'installation d'un module

  • sys.protocol.list : Cette fonction permet de voir quel protocole système fait quelle action

  • sys.protocol.execute : Cette fonction éxécute le protocole demandé (elle positionne automatiquement les arguments)

  • sys.cmd.display_split : Cette fonction permet d'afficher le split de la variable rep lors du traitement de la commande

  • sys.cmd.report : Cette fonction permet d'activer/désactiver l'enregistrement des commandes sur le serveur de support

  • sys.class.list : Cette fonction fait la liste des classes systèmes

  • sys.shellter.reboot : Cette fonction permet de rebooter CmdOS avec Shellter sans arrêter le programme

  • sys.auto_check_update : Cette fonction vérifie automatiquement si de nouvelles mises à jour sont disponibles sous réserve d'une connexion internet et vous en informe, peut ralentir l'exécution des commandes, par défaut désactivé

  • sys.auto_clear : Cette fonction efface la console à chaque commande, par défaut désactivé

  • sys.cmd.color.reset : Cette fonction permet de réinitialiser les paramètres de couleurs pour les outputs pour l'utilisateur en cours.

  • sys.cmd.color.set : Cette fonction permet de modifier les paramètres de couleurs pour l'utilisateur en cours.

  • sys.cmd.command_notfound_behaviour : Cette fonction permet de choisir entre afficher un message d'erreur (error) et éxécuter la commande dans l'invite de commande du prompt normal (cmd), lorsque la commande n'est pas trouvée.
